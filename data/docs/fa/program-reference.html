بـستا Program Reference

This document will introduce the بـستا user interface and its main functionalities.

Main Window

This section covers the elements constituting the بـستا main window.

Title

The main program window's title contains the name of the program and the project filename which you are currently working on.

Please note that the asterisk after the filename means that the saved project is not up-to-date with the current situation and should be saved again.

Menu

File Menu

Allows opening and saving projects, images, and videos; printing images and quitting the program.

  • New Project: creates a new empty project.
  • Open Project...: loads a previously saved project.
  • Recent Projects...: contains a list of recent projects for a fast reloading.
  • Save Project: saves the current project.
  • Save Project As...: opens a file dialog which helps to choose the name and the path of the file to be saved.
  • Open Image or Video...: opens a file dialog to open an image or video file.
  • Copy and Verify...: a tool to quickly and reliably acquire evidence files.
  • Convert DVR...: open the tool to convert proprietary DVR videos into standard formats.
  • DVR Screen Capture...: open the tool to capture the screen, for proprietary DVR video which cannot be converted by other means.
  • Advanced File info...: opens a dialog containing basic and advanced information about the loaded image or video file.
  • Export video...: a feature to export the loaded video.
  • Export PDF...: a feature to export the selected frames into a pdf file.
  • Print...: prints the image or frame displayed on the currently active Viewer.
  • Page Setup...: setups the page to be printed.
  • Print Preview: displays how the project will look when printed.
  • Exit: exits from the program.

Project Menu

Project related functions, such us project properties configuration and report generation.

  • Generate Report: generate a detailed report in html format with the description of the full processing and including the bookmark images and notes; the report is automatically opened in the default system browser
  • Export Bookmarks Images: export all the bookmark images in the default format
  • Save Snapshot: saves the currently displayed image and the current status of the project
  • Open Project Folder: opens the file Explorer in the folder containing snapshots images and projects

View Menu

Allows changing the aspect of the graphical user interface of بـستا.

  • Filters: shows/hides the corresponding panel.
  • Assistant: shows/hides the corresponding panel.
  • Tools: shows/hides the corresponding panel.
  • Filter Settings: shows/hides the corresponding panel.
  • Player: shows/hides the corresponding panel.
  • Audio: shows/hides the corresponding panel within the Player.
  • History: shows/hides the corresponding panel.
  • Bookmarks: shows/hides the corresponding panel.
  • Toggle Full Screen Viewer: shows/hides all panels.
  • Toggle Full Screen Viewer with Player: shows/hides all panels but the Player.
  • Open Log Folder: opens the folder containing the logs.
  • Project Properties...: loads the dialog to modify project settings
  • Reset Layout and Options: will restore program layout and settings to the default values. Restart the program to let them take effect.

  • About بـستا: shows license and version information.

Project Properties

In the Project properties you can add some useful information about your project, which will complete the automatically generated report.

  • Case: the name of the case you are working on.
  • Author: your name.
  • Description: any useful info, text, or narrative, you'd like to add about the case.
  • Report Image Width: with of the images in pixels in the report. This will not influence the actual size of the exported images, but only their display size in the report. Leave setting at 0 to use the default width.

Program Options

The program options define general settings of the software.

  • Project Auto-Recovery (enabled by default): once enabled, if the program is forced to stop processing (due maybe to a crash or power failure), بـستا will be able to restart from the last operation. In this case, when started, the program will warn the user that it detected an improper exit and will offer the possibility to reload the interrupted project.
  • Language: lets you choose software localization, the default value uses your system settings if it's a supported language, otherwise it is English.
  • Default Image Format: sets the default image format for snapshots, exported bookmarks and filters that save images.
  • Default Image Quality: sets the default image quality for snapshots, exported bookmarks and filters that save images.
  • Default Video Format: sets the default video format and codec for filters that save videos. Choose avi - MSPEMG4v2for maximum compatibility and avi - Raw Video for the best quality. In the second case please note that the generated video file may become huge if you need more than few seconds of footage.
  • Default Quality for H264 Codec: sets the default video quality in case the chosen codec for transcoding is H264.
  • Log: enables or disables log file creation.
  • Snapshot Mode: sets the operation to do upon clicking the Snapshot button.
  • Snapshot Data: sets the content of the snapshot.
  • HiDPI Mode: enables or disables the HiDPI Mode (interpolation of the user interface elements to preserve the layout on high-resolution screens).
  • Color Style: sets the theme of the interface.
  • Font Size: sets the font size for the user's text.
  • Use FFMS with Audio as Default: enables or disables FFMS with Audio as the default video engine.
  • Show Filter Id on History and Report: enables or disables a numerical ID for the filters in the History and the generated Report.
  • Auto Load Timestamps: enables or disables the automatic timestamp loading (if available).
  • Filters Panel Alignment: sets the desired option for the Filters Panel Alignment.
  • Remind me about Expiration: sets the desired frequency of the reminders for the Software Maintenance and Support expiration.
  • *Auto Add Windows Date and Time to Notes: enables or disable the adding of the Windows date and time to Notes.

Tool Bar

Under the menu there is the program's toolbar: by clicking on the icons, you can, in the following order:

  • New: creates a new empty document;
  • Open: opens a saved project;
  • Save: saves the current project;
  • Open Image or Video...: opens a file dialog to open an image or video file.

Status Bar

The Status Bar on the bottom of the main window contains important information about the current state of بـستا. It is normally in "Ready" state, but it becomes "Busy" as soon as بـستا receives a command. Once done, the status bar returns in "Ready" state. In the case of time-consuming filters, progress percentage is shown.

Viewer Window

The Viewer is the window in the center of the screen. It shows the frame which we are currently working on.

Each chain in the workspace has its Viewer: when closing the Viewer, the chain will be deleted from history.

The image inside the Viewer shows the effect of the filter currently selected in the history and of the previous ones. Thus, the review of the processing steps is very simple and easy to understand: you have just to click on the desired filter.

It is possible to drag different files into the Viewer. You can drag an image, a video or an .afp project file. It is also possible to drag multiple images (creating a Sequence Loader) or multiple videos (creating one chain for each video). Dragging a folder is also supported and is equal to dragging its contents. The new chains with the dragged files appear at the position that is currently selected in the History. If you want to place them at the end, just clear the selection by clicking on the empty space at the bottom of the History.

Filters Panel

The Filters panel, located on the left side of the main window, contains all the filters you can apply to an image or video. They are divided in different categories, according with the type of action they perform. By double-clicking on a category, the complete list of filters will be opened.

To select one of them and add it to the current chain in the history, just single-click the desired filter. The new filter will be immediately added after the last selected filter. On the other hand Loaders always create a new chain.

Please note that all filters can be applied to images, sequences of images and videos. However, there are cases in which the output of a filter returns the input image unchanged. This occurs mainly when you apply to images filters that need more than one frame to work properly. The categories of filters are the following.

Note: depending on your version of بـستا some of the following groups may not be available. If you require the additional filters showed, please inquire about upgrading your version.

Load group

This type of filters allows loading images, videos or sequences from files. They allow selecting the file by typing a path, using the system dialogs, or by dragging and dropping it from the desktop or any Explorer window.

Since revision 18800, new Loaders are created at the position that is currently selected in the History. If you want to place them at the end, just clear the selection by clicking on the empty space at the bottom of the History.

Write group

Writers allow saving the output of a filter by writing the result of processing (image or video) to one or more user defined files. Please note that, once saved, if you add any filter after the writer, it will work on the saved file. Therefore, we may say that a writer is a sort of loader for some aspects.

Differently from all other filters, writers have to be updated manually. If some change is made on any filter in the same chain before the writer, the output will not be overwritten until the user pushes again the "Apply" button on the writer's toolbar. A writer can be useful in many ways:

  • it allows exporting the results of processing by generating a file containing the processed data;
  • it allows a non-sequential access to those videos that use particular codecs that prevent it, as described in the FAQ section;
  • it allows making a sort of explicit buffering. Each frame is calculated in real-time and sometimes the amount of work may be too heavy for the CPU; in this case, the playing slows down. A writer may come handy in this situation because it saves results of previous filters, so they don't need to be re-calculated on the fly. Every new filter added to the chain, after the writer, will continue to be computed in real-time.

Select Frames group

When using this kind of filters, you can select one or more frames of interest from the input file. It is possible to use any media as input, but a selector will have no effect on a single image. Generally, they change the length of a video, but they don't modify its frames. If you choose, for example, frames from 10 to 20, in the resulting output media the frame number 0 will correspond to frame 10 of the input media. To avoid mistakes, you can configure selectors only when "Show Input" mode is selected, i.e. when you are displaying the input and not the output image.

Deinterlacing group

These filters change an interlaced video into a progressive one.

Edit group

This kind of filter implements basic image processing functionality involving geometric transformations. Generally:

  • they separately work on single frames;
  • they can modify the image characteristics (size, number of channels, color depth);
  • they don't modify the video length (number of frames).

Channels group

These filters allow conversion, extraction and replacement of color channels.

Adjust group

These filters are useful to tune the pixel intensity of a specific region of the image. In general:

  • they separately work on single frames;
  • they don't modify the image characteristics (size, number of channels, color depth);
  • they don't modify the video length (number of frames).

Without going into details, we can say that all digital images are represented by a sequence of numbers between lower and upper limits. In the 8-bit format these limits are 0 and 255. For every pixel there is a numeric value of this type. For black and white images, the value 0 corresponds to a black pixel, while the value 255 stands for white. Color images are slightly different because they are made of three channels (red, green, blue). Each channel can be represented as a black and white image that contains data about the intensity of the color, pixel by pixel. This decomposition is done in such a way that, putting together the three components again, we will get the original image. Filters of the "Adjust" category map separately input image values to other values of the output image, for every single pixel, following a certain table ("look-up table").

Extract group

Edges extraction and separation of some information of the image . Fourier, for example removes periodic noise, background and interferences in the Frequency domain. Useful for latent fingerprints, capture artifacts and electromagnetic interferences in videos.

Verify group

These filters allow checking metadata of the image or verifying the hash code for tracing the originality of images. They don't process the image at all; they simply provide some useful information on the source file and also display the results in the report group.

Measure group

These filters allow taking different kind of measures on the images.

Sharpen group

The filters of this category don't change the image characteristics; conversely its contents are processed to emphasize details and borders of the present objects. They can make appear the finest details, but if you go further, they can also raise noise level.

Denoise group

These filters have the opposite effect of the sharpening ones. They are used to reduce the image noise level. This is usually noticed as granularity or pixels with the wrong color. You could need to use these filters, when you have a digital camera shot taken in a low light condition.

Deblurring group

This category contains various filters for image restoration and enhancement. They are used for solving very specific problems, such as motion blur or compression artifacts. They don't modify image characteristics or video length. Every single frame is elaborated separately.

Stabilization group

Filters in this category perform different kind of stabilization. They provide results only for videos or sequences of more images. The stabilization filters may either keep a region of interest steady in the image or stabilize the overall scene.

Integrate group

In this category we can find the most powerful and CPU-intensive filters. They work on sequences and videos and their output could be very different from the original file. In general the result of the processing is an image which "sums up" all details and information present in every single frame.

Presentation group

These filters provide some utilities for better presenting the final result for the processing, for example putting side by side the original and the processed image or hide sensitive details like witnesses faces or license plates of vehicles unrelated to the case.

Assistant Panel

The Assistant panel provides a tool which is something between a scripting system and wizard, allowing the user to go through a series of standardized steps in order to follow a proper workflow. For example, an agency can implement in an Assistant script its SOPs (standard operating procedures) in order to standardize the users workflow for specific situations. بـستا includes several workflows to be used as a starting point for some common situations.

The different workflows can be customized or added to for adaptation to individual or organizational preferences and needs. The panel can be displayed or hidden using the View menu or keyboard shortcut "Alt + A".

Workflow scripts can be selected using the drop down menu under the toolbar in the Assistant panel.

The pre-defined workflows and scripts supplied are as follows:

  • Assistant home: welcome message displayed when first accessing the assistant panel.
  • Assistant tutorial: tutorial to assist with general use of the assistant and for creating custom workflows and scripts. Please use this if you want to understand how to create and modify a workflow.
  • Enhancing a face from a video: enhancing the face of a subject putting together several frames of a video.
  • Enhancing a fingerprint: correcting the most common issues to enhance a fingerprint from a single image.
  • Enhancing a license plate from a single image: enhancing a vehicle license plate from a static image or frame.
  • Enhancing a license plate from a video: enhancing a vehicle license plate from a video or image sequence where it is visible on multiple frames.
  • Favorite filters: list of regularly used filters which can be edited to include personal filter preferences.
  • General workflow (cheat sheet): this includes all the filters of بـستا in a single step. It is useful as a reference to know which filter to apply in every step of the processing.
  • General workflow (guided): this is a very long workflow that guides the user through the application of all the filters in بـستا, helping the user understand the proper filter for every situation and the optimal order in which to put them.
  • Measuring the height of a person: measuring the height of a subject in a video or image.
  • Prepare for social media release: recommended for instances in which images will be released to the general public, for example on social media, with possible redactions, annotations and other presentation improvements.
  • Redacting a body worn camera video: This workflow is recommended for removing sensitive material from body worn video.

The panel itself, located in the bottom left corner by default, has a toolbar at the top allowing for the creation of new scripts or navigating between workflow steps.

  • Create new script: this command, represented by an icon with a blank page, allows you to create a new script should you want to create a custom workflow.
  • Duplicate the current script: this command, represented by an icon with two overlapping pages, allows you to duplicate the currently loaded script should you want to alter the workflow slightly but keep some of the steps.
  • Edit the current script: this command, represented by an icon with a pencil, allows you to edit the currently loaded script should you want to alter any of the filters or comments used in the workflow.
  • Rename the current script: this command, represented by an icon with two letters, allows you to rename the currently loaded script.
  • Delete the current script: this command, represented by an icon red cross, allows you delete the currently loaded script.
  • Open the Assistant scripts folder: this command, represented by an icon with a folder, allows you to navigate to the folder in which the various Assistant scripts are stored.
  • Refresh the current assistant script and the script list: this command icon, represented by an icon with a looped arrow, allows you to refresh the currently loaded script and the list of scripts should you have recently updated a workflow.
  • Go to the previous step in the current script: this command, represented by an icon with a green arrow pointing backwards, allows you to return to the previous step in the workflow of a currently loaded script.
  • Go to the next step in the current script: this command, represented by an icon with a green arrow pointing forwards, allows you to jump to the next step in the workflow of a currently loaded script.

To create a script, filters from بـستا must be added by name in a particular format. Filters should be added in square brackets and comments or actions in round brackets. A tutorial is included to assist with creating new scripts.

Tools Panel

The tools panel contains all the tools that allow you to interact with the dislayed image or with the filters. Thus, they don't actually process the data. You can activate a specific tool clicking on the tool at the top of the panel. Below is a description of various tools. Every tool condition the behavior of mouse command on the viewer.

Inspector

The Inspector tool allows us to know the value of an image pixel. When the mouse is moving around the Viewer window, the coordinates x and y in the tool will vary accordingly. So will the R, G, B values of the pixel indicated below the mouse pointer.

If the Inspector is active, you can move around the image by dragging it into the Viewer.

The inspector displays the following values related to the pixel where the mouse is currently over:

  • x: top-0 based horizontal coordinate of the pixel;
  • y: left-0 based vertical coordinate of the pixel;
  • R: red channel pixel value (not valid for grays scale images);
  • G: green channel pixel value (not valid for gray scale images);
  • B: blue channel pixel value (not valid for gray scale images);
  • Y: intensity value of the pixel (corresponding to the value of the pixel for gray scale image and the gray scale converted pixel value for color images).

In the Details box, the additional information on the currently displayed frame is shown.

  • Position: current frame position in a video (always 0 for a single image);
  • Width: horizontal resolution of the current image in pixels;
  • Height: vertical resolution of the current image in pixels;
  • Type: color (3 channels) or gray scale (1 channel).

Zoom

The Zoom tool allows to magnify or to scale down the image. When selected, if you move the mouse pointer over the Viewer, the pointer will become a magnifying glass and you will be able to zoom on the area of interest.

  • In the Zoom combo box some predefined zoom values may be selected.
  • With the + and - buttons you can raise or lower the zoom factor.
  • The button 100 applies a zoom factor so that every pixel of the screen corresponds to a pixel of the image (1:1 scale);
  • The Fit Window checkbox resizes the image so that it will match the window's size.
  • Below those controls you will find a little preview and a selection rectangle over it. In the Viewer is displayed the contents of the selection rectangle and vice versa. Just above the preview area there is a slider; the more it is on the right (high zoom values), the smaller the rectangle is. Dragging the selection rectangle in the preview navigator will automatically shift the image in the Viewer, so it will be possible to perform image drag & drop.

Please note that, if the Zoom tool is selected and you left-click on the image in the Viewer, you will raise the zoom factor, while with the right button you will lower it. By selecting a rectangle on the Viewer you will zoom on that area.

Note: the zoom panel does not apply any interpolation to the displayed image, it applies a simple pixel replication or decimation, depending on the zoom factor.

Note: you can always zoom in and out the current image even if the zoom tool is not active. Simply go with the mouse over the viewer and use the mouse wheel.

Selector

The Selector tool allows selecting an area of the image. When selected, if you move the mouse pointer over the Viewer, the pointer will become a cross and you will be able to select an area of the image. Just keep pressed the left mouse button while moving the mouse, to draw the selection rectangle. Once done, the tool panel will show the selected coordinates.

  • By clicking on Select All, the selection will be expanded and will cover the whole image.
  • By clicking on Deselect, the selection will be disabled. Please note that, if you change your mind at any time, you can achieve the same result by clicking on a point of the image that is outside of the selection rectangle.
  • The Zoom button will automatically apply the maximum possible magnification to the selected area.

Please note that, once selected, it is possible to move the selection rectangle around: just place the mouse pointer on the center of it, and then drag it as you like. If you want to modify the rectangle's size, just place the mouse on its corners and drag it.

The Selector tool is generally used to configure a selection inside a filter (generally those one which contain the tab Select in the GUI).

Ruler

The Ruler tool is used to define the distance between the objects in an image. When selected, if you move the mouse pointer over the Viewer, the pointer will become a cross and you will be able to select any couple of points on the image. As the segment of interest has been selected, the tool panel will show its "Point 1" and "Point 2" coordinates, its pixel "Length" and its "Angle" with respect to the horizontal axis. If you want to change the selected points afterwards, just drag them wherever you want; while, the selected segment will be disabled by clicking on a point of the image.

The Ruler tool is used by some filters (for example, Motion Deblurring or Measure 1d/2d/3d) in order to automatically configure parameters related to distance and angles.

Quadrilateral

When the Quadrilateral tool is selected, the mouse pointer will become a cross moving it over the Viewer and you can select the four vertices of the polygon that will be drawn on the image. When you mark the coordinates, they will appear in the Point 1, Point 2, Point 3, Point 4 boxes of the tool panel. If you want to change the selected points afterwards, just drag them wherever you want; while, by clicking outside the selected polygon, it will be disabled.

The Points is used by some filters (for example Perspective Correction) to automatically configure parameters related to some selection areas.

Point

The Point tool allows selecting a single point in the image.

The Point is used by some filters to configure single point coordinates.

Histogram

The histogram panel shows the histogram of the currently displayed image and some details on the statistical distribution of pixel values. The image histogram is a graphical representation of the tonal distribution in a digital image. The horizontal axis of the graph represents the tonal variations (from 0 to 255), while the vertical axis represents the number of pixels in that particular tone.

With the Channels option you can choose what channel(s) to plot on the histogram:

  • RGB: plots the histogram of all the three channels together;
  • Red: plots the histogram of the red channel;
  • Green: plots the histogram of the green channel;
  • Blue: plots the histogram of the blue channel;
  • Y: plots the histogram of the intensity value (corresponding to the image itself for gray scale images and the gray scale converted image for color images).

Under the histogram you will find, for the selected channel(s), the following statistics:

  • Min: minimum value of the image (0-255);
  • Max: maximum value of the image (0-255);
  • Mean: average value of the image (0-255);
  • StdDev: standard deviation of image values.

At the top of the panel you can select the box Show Saturated Pixels. If you check it, all pixels whose value is 0 (R = G = B = 0) will be colored in blue and all pixels whose value is saturated (R = G = B = 255) will be colored in red. This is very useful to verify you are not applying excessive enhancements causing loss of information.

Filter Settings Panel

The filter settings panel allows changing the configuration of the currently active filter.

In the title part of the Filter Settings panel you can see whether you are display the input or the output of the filter you are configuring and if the currently set parameters have been applied.

On the tool bar of the filter settings window you can find the following buttons (their availability changes depending on the filter type):

  • On / Off: allows temporarily disabling of the filter; the configuration is not lost, but until you enable it again, it will be simply bypassed in the processing change, as if it was not present there;
  • Auto-Apply: if checked the filter is updated every time you change some parameter. If disabled you will need to click Apply every time you change a parameter to commit the change;
  • Help: opens the filter reference to get more information about the current filter type. Not that you can get some information about the parameters passing over them with them mouse;
  • Reset: reset filter parameters to the default value;
  • Show Input: shows the input image for the current filter;
  • Apply: apply the parameters set and shows the output.

In order to activate the Filter Settings window for a specific filter, you must double click on it on the History panel. If you want to configure a filter while looking at the output of another filter (that must be after the one you are configuring), you can do a single click on it on the History tree.

Setting Filters Selection

An option that all local filters have in common the Selection tab (usually the last one on the right). With the Selection tab you can set the current filter to act on a specific region of interest in the image. These are the available options.

Whole Image

With the option Whole Image the filter is applied to the whole image

Static Region

The filter is applied only to the specific region selected, which is kept fixed in all the frames.

Static Tracking

The filter is applied only to the selected area of interest, tracking the object of interest in all the frames. In order to set the object to track, find a frame where you see it better, set a selection around it and click Set on the Selection tab. For every frame, Static Tracking will try to find the best matches the reference selection taken on the reference frame. This type of tracking is best for non-animated subjects.

If you set only one reference frame, it will be used as reference for the tracking in all the other frames. If you set more than one reference frame, the tracking will start on the first reference frame and will stop with the last reference frame. You can also set intermediate reference selections.

Dynamic Tracking

The filter is applied only to the selected region, tracking the selected area of interest in all the frames. In order to set the object to track, move to a frame where you see it best and set a selection around it. Then click Set on the Selection tab. Dynamic Tracking will attempt to find the area of the image in each frame the best matches the reference selection on the previous frame. Dynamic Tracking works differently from Static Tracking as the reference selection will be updated after each frame. The best results can be obtained if the reference frame is set to the first frame of the video where the object of interest appears.

If you set only one reference frame, it will be used as reference for the tracking in all the other frames. If you set more than one reference frame, the tracking will start on the first reference frame and will stop with the last reference frame. You can also set intermediate reference selections.

For significant movement, the more reference frames specified, the better the possible result.

Manual Tracking

Manual Tracking allows the user to select manually different selections of different frames. The selections on intermediate frames will be calculated interpolating the position and the size of set selections on closest frames.

Suggestions for different tracking types

If you want to keep the scene steady around a moving object it is better to use either Static Tracking or Dynamic Tracking.

Static Tracking is usually more precise but less robust then Dynamic Tracking. It tracks the object very accurately if it's not changing too much from frame to frame. For example usually Static Tracking is very effective on shaking license plates or other inanimate objects. If the appearance of the license plate is changing too much, for example because the car is coming closer to the camera, then you'll need to use Dynamic Tracking. Also if you need to track people or faces, Dynamic Tracking is usually the better choice.

If the target to track is not present in all the frames, you first want to work only on a part of the video. It is helpful to use a filter from the Select Frames group and select the frames where the target is present prior to using a filter with Dynamic Tracking selected.

Manual Tracking is a better option when the scene is static but you want to follow some subject moving in it.

For all the tracking modes, with:

  • Set: you will erase all current reference frames and set the current as the only one;
  • Add: you will add the current frame and selection to the reference frames
  • Remove: will delete the current reference frame.

Clicking on the items on the list, will move the position of the video to the reference frame and will select the corresponding reference area.

The behavior of the various tracking mode changes if there is one or more reference selection. If there is only one reference selection, that will be used to process the whole video; if there are two or more reference selections, the tracking will be properly applied only in frames between the first and the last reference selection.

For the filters Crop and Local Stabilization it's recommended to use only one single reference frame or to keep constant the size of the selections if using more than one reference.

History Panel

The History shows all the filters that have been added in the current project. Filters are grouped in Chains. The first filter of every chain is either a loader (from the “Load” category) or a mixer (from the “Link” category). Any filter with the exception of the first one receives the image generated from the filter it has in input and send the result of the processing to the filter (if any) in output.

Since revision 18800, it is possible to organize chains in folders. If a project that uses folders is opened in an older version of بـستا, only the chains will appear one after the other at the same level, and saving the project will destroy the folder structure. Folders can be added to an empty project by clicking on the “New Folder” link on the History panel. A project containing only empty folders can be saved and reopened, but will give an error in previous versions of بـستا. It is not yet possible to duplicate folders or to nest them in multiple levels.

Chains and folders can be renamed by double-clicking on the name. To confirm the change, press ENTER or click outside; to cancel it, press ESC. It is also possible to right-click on the name and select “Rename Chain” / “Rename Folder” on the drop-down menu.

Chains and folders can be collapsed and expanded by clicking on the and buttons on the tree view. Empty folders don't have buttons. The tree is automatically expanded to visualize the current filter.

Filters, chains and folders can be deleted by selecting them and pressing the DELETE key or by right-clicking them and selecting “Delete” from the drop-down menu. For folders, a dialog box will appear asking if you want to delete the contents as well or to move them outside.

Chain mixers (from the “Link” category) must be positioned after their inputs. Only the filters before the mixer will show as available inputs in the Filter Settings panel. Attempting to move a mixer before its inputs, or an input after the mixer, will produce an error message.

Activating a filter

To display the output of a filter, you just need to click on it with the left button of the mouse. With a double click you will also activate the Filter Settings panel for that filter. If you want to adjust the settings of a filter while displaying the output of another one, first double-click on the filter to adjust and then single-click on the filter to display.

Filters can be disabled and enabled by clicking on the check box next to their name. The first filter of a chain cannot be disabled, and the check box is therefore dimmed.

Context menu

Right clicking on the history panel will activate the context menu. Available actions will depend on the clicked item: first filter of a chain, other filters, chain, folder or empty space. These are the available commands:

  • Cut: cuts the selected item (the item remains visible at the original position until pasted);
  • Copy: copies the selected item to memory (not yet supported for folders);
  • Paste and Replace: substitutes the selected item with the one that has been previously cut or copied (this menu entry is auto-enabled only when an item of the same type is available);
  • Paste Before: inserts the item in the position, immediately before the current selected one;
  • Paste After: inserts the item in the position, immediately after the current selected one;
  • Paste Inside: inserts a chain into the selected folder, at the first position;
  • Move Outside: moves the selected chain out of its folder, after it;
  • Delete: deletes the selected item;
  • New Folder: creates an empty folder before the selected chain (if clicking on a chain) or at the end (if clicking on the empty space after the last chain);
  • Rename Chain: allows renaming the selected chain;
  • Rename Folder: allows renaming the selected folder.

Dragging an item

It is also possible to use the drag and drop function. That means that you can drag an item to any position inside the history. A marker will appear showing the position the dragged item will go to. To cancel a drag and drop operation, move the mouse cursor out of the History panel before releasing the mouse button.

The default operation (move or copy) depends on the type of action, as explained below. Holding down the CTRL key will force a copy, and holding down the SHIFT key will force a move. Copying a chain will also clone all the filters it contains. The shape of the cursor changes to indicate the type of operation:

The item is being moved The item is being copied The operation is not allowed

The possible filter operations are:

  • Dragging a loader in the empty space after the last chain will create a new chain with a copy of that loader.
  • Dragging a non-loader filter onto another filter of the same chain will move the filter after the target.
  • Dragging a non-loader filter onto a filter of a different chain will copy the filter after the target (hold down SHIFT to move instead).
  • Dragging a filter on the Bookmarks panel will create a new bookmark for the filter at the current position in the video.

The possible chain operations are:

  • Dragging a chain at the top of the panel will move it at the beginning, outside any folder.
  • Dragging a chain in the empty space after the last chain will copy it at the end, outside any folder (hold down SHIFT to move instead).
  • Dragging a chain onto a folder will move it inside that folder, at the first position.
  • Dragging a chain onto another chain (or its contents) will move it after the target, at the same folder level.

The possible folder operations are:

  • Dragging a folder at the top of the panel will move it at the beginning.
  • Dragging a folder in the empty space after the last chain while holding down the SHIFT key will move it at the end.
  • Dragging a folder onto another folder (or its contents) will move it after the target.

Dragging a file

It is possible to drag different files into the History. You can drag an image, a video or an .afp project file. It is also possible to drag multiple images (creating a Sequence Loader) or multiple videos (creating one chain for each video). Dragging a folder is also supported and is equal to dragging its contents.

The possible file operations are:

  • Dragging an .afp project file closes the current project and opens the dragged one.
  • Dragging files over the text label of a loader can either replace the loader or create a new chain after it. A dialog box will appear, asking the user what to do. Note that the behavior has changed in revision 18800: previous versions of بـستا only allowed to replace the loader.
  • Dragging files over other parts of the History creates new loaders at the cursor position, indicated by a marker. Files can be dragged at the top, over a chain (creating the new loaders after it, at the same folder level), over a folder (creating the new loaders inside, at the first position) or at the bottom.

Folder management

New folders can be created by right clicking on the desired position and selecting “New Folder” from the drop-down menu. The folder will appear before the selected item. If you want to create a folder at the last position, right-click on the empty space after the last chain.

To move a chain inside a folder, you can either:

  • Cut / Copy and Paste Inside,
  • Cut / Copy and paste onto another chain that is already inside the folder (using Paste and Replace, Paste Before or Paste After),
  • Drag onto the folder,
  • Drag onto another chain that is already inside the folder.

To move a chain out of a folder, you can either:

  • Cut / Copy and paste onto another chain that is outside the folder (using Paste and Replace, Paste Before or Paste After),
  • Cut / Copy and Paste Before / Paste After the folder,
  • Cut / Copy and Paste onto the empty space at the bottom of the panel,
  • Select “Move Outside” from the drop-down menu,
  • Drag onto another chain that is outside the folder,
  • Drag at the top of the panel,
  • Drag in the empty space after the last chain (hold SHIFT to move).

Bookmarks Panel

The bookmarks panel allows to store some filter positions in order to easily switch between salient points in your project.

You can add a new bookmark:

  • dragging a filter from the History to the Bookmarks panel;
  • clicking with the right mouse button on the Bookmarks panel and then selecting New Bookmark;
  • clicking the Add Bookmark button on the Player panel.

A bookmark stores the following info:

  • Filter: when the bookmark is selected it will activate the corresponding filter
  • Position: when the bookmark is selected it will seek the defined position
  • Name: you can define a custom name for the bookmark;
  • Description: you can define a description (or simply add a note to your bookmark).

In order to set name or description of a bookmark click with the right mouse button on the bookmark and select Edit Name or Edit Description. Selecting Delete in the bookmark menu you can delete it.

When you generate the report, all the bookmark images will be included with the accompanying names and descriptions. You can change the format of the exported images with View / Program Options... / Default Image Format

Bookmarks relative to filters that have been deleted, will be removed, too.

Player Panel

The player panel allows to control playback of a video and to move between frames.

The player panel has the following controls, starting from the left. Most of them will be disabled for single images.

  • Play / Pause: start or pause the playback.
  • Stop: pause the playback.
  • Previous: go to the previous frame.
  • Next: go to the next frame.
  • Snapshot: save the snapshot of the currently displayed image and relative project.
  • Open Project Folder: open the snapshots folder.
  • Add Bookmark: add a new bookmark with the active filter on the current position.
  • Position: text box display playing position (current frame number) and playing time (hh::mm::ss.msec). If the video is not playing, you can move on by inserting the desired position: if you give an integer number (ex. 10) you will move to the 10th frame, while if you give a decimal number (ex. 10.0) you will move to the nearest frame to the 10th second of playing (the format, 00.10.0, can be used too).
  • Loop: if active, when the end of the footage is reached, playing automatically restarts (otherwise stops).
  • Special Seek Controls: the special seek buttons allows moving in the video by a custom amount of time or frames. You can configure, for example, that every time you press the Special Next button you go on by 10 frames, or 3 minutes, using the proper amount and unit controls.
  • Frame Rate Controls: the slider allow to set the target frame rate. It may actually be lower than the desired one, if the filters to process are too heavy. By clicking on "Reset Fps", it will reset fps to the original value.

The frame rate mode (on the right side) allows selecting if you want to use the frame rate slider to adjust the frames per second or like a jog control. The available modes are:

  • Speed: a frame rate multipler/reducer, relatively to the frame rate in use in the filter currently selected (default);
  • Seek: if you leave the slider on the center the video stays stopped, if you move it on the right it will go forward, on the left it will play backward. The playback speed it proportional to the distance of the cursor form the center. Please note that all frames will be played;
  • Skip: similar to Seek, but frames will be skipped; it's faster than Seek, but excessively large steps may obstacle the vision of some sudden event in the video.

The Position slider, on the bottom of the Player panel will let you move on the range of frames of the video. On the left you can read start frame and start time and on the right side stop frame and stop time.

Audio Panel

This is part of the Player Panel which allows previewing of the audio track associated with a video.

Different colors on the preview correspond to different channels of the audio track. For example, red to the left channel and blue to the right channel in a stereo (2-channel) audio track.

Zooming the audio track

Usually, the audio sampling frequency is so high that a sample-by-sample plot becomes unreadable at lower zoom levels; in those cases, the envelope* plot is used instead.

There are several ways to zoom into the preview in order to see more audio track details.

  • Right-clicking over the plot activates the context menu. There are zooming actions like zoom in, out or reset, both horizontally and vertically.
  • Holding CTRL enables zooming horizontally using the mouse.

  • Holding SHIFT enables zooming vertically using the mouse.

    • Left-click or wheel up will zoom in.
    • Right-click or wheel down will zoom out.

* Envelope, the outer shape of a signal plot, presented here is the root-mean-square (RMS) envelope, determined using a sliding window over multiple samples. It gives a measure of the power of the signal.

Navigating through the audio track

The Audio playhead automatically links with the video playhead.

A Left-Click of the mouse over the Audio visualization causes the video to seek to the frame corresponding to the clicked point.

It is possible to scroll the preview with the mouse wheel. Wheel up will seek frames forward and wheel down will seek frames backward.

Similarly, the audio track can be navigated forward or backward by mouse dragging.

Annotation Ranges Panel

This is part of the Player Panel which allows previewing of annotation ranges associated with the currently configured Annotate filter. The Annotation Ranges Panel substitutes the Audio Panel when an Annotate filter is being configured.

Understanding the preview

The preview eases handling multiple annotations. Colored lines indicate ranges of frames to which annotations are applied. Every line is related to a single annotation.

The color of each line is determined by the color the user sets for the annotation object. This is a quick and easy way to distinguish different annotations straight from the player panel.

The vertical order with which lines stack over each other reflects the current layering of annotations. For example, if you bring to front an annotation, its line will move up to the top. That’s a very effective and simple way to know which is the current layering of all annotations in every frame.

When you click on an annotation object, you will notice two vertical segments at both ends of a line. It’s a way to quickly show where the currently selected object appears and disappears from the video. This is especially useful when you have many annotations, and therefore many lines in the preview.

Navigating through the video track

The panel playhead automatically links with the video playhead.

A Left-Click of the mouse over the panel causes the video to seek to the frame corresponding to the clicked point.

It is possible to scroll the preview with the mouse wheel. Wheel up will seek frames forward and wheel down will seek frames backward.

Similarly, the video can be navigated forward or backward by mouse dragging.

Commandline parameters

Usage: five.exe [/i <str>] [/o <str>] [/c <str>] [/pgeadi]

  • /i <str>, --input_path=<str>: path of the image/video file or the folder to load
  • /o <str>, --output_path=<str>: outputfilename to generate
  • /c <str>, --command=<str>: command to call (load|convert|screencap); when "convert" is used and it's not possible to preserve the original encoding then the file will be transcoded with h264
  • /pgeadi, --plugin_geadi: run as plugin for Griffeye Analyze DI
